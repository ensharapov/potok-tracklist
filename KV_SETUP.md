# Настройка хранилища для синхронизации прогресса

Для постоянного хранения прогресса между устройствами нужно настроить хранилище через Vercel Marketplace.

## Шаги настройки:

1. **Открой Vercel Dashboard** → выбери свой проект
2. **Перейди в раздел "Storage"** → нажми "Create Database"
3. **В модальном окне выбери "Marketplace Database Providers"**
4. **Выбери "Upstash"** (или "Redis"):
   - Upstash предоставляет Serverless Redis/KV
   - Нажми на "Upstash" → "Continue"
   - Следуй инструкциям для создания базы
   - Выбери регион (ближайший к твоим пользователям)
   - Назови базу, например: `potok-progress`

5. **Подключи базу к проекту:**
   - После создания Upstash автоматически подключится к проекту
   - Vercel создаст переменные окружения для подключения
   - Они автоматически доступны в serverless functions

6. **Обнови код для работы с Upstash:**
   - Нужно будет использовать `@upstash/redis` вместо `@vercel/kv`
   - Или можно оставить текущий код - он будет работать с fallback на in-memory

7. **Передеплой проект:**
   ```bash
   git push
   ```
   Vercel автоматически пересоберёт проект с новыми переменными окружения

## Проверка работы:

После настройки прогресс будет сохраняться в KV и синхронизироваться между всеми устройствами пользователя.

## Альтернатива (если не хочешь настраивать KV):

Можно использовать другие БД:
- **Supabase** (PostgreSQL) - бесплатный тариф
- **PlanetScale** (MySQL) - бесплатный тариф  
- **Upstash Redis** - бесплатный тариф

Но Vercel KV - самый простой вариант для начала.
